# PR Review Template (AI Reviewer)

## 1. Summary Quality
- Is the PR clearly described?
- Are linked issues referenced?
- Is the purpose explained?

## 2. Code Quality
- Naming conventions consistent?
- No duplication?
- Functions/classes small & single-purpose?
- Correct error handling?
- Applies SOLID principles?

## 3. Testing
- Are appropriate tests added?
- Tests deterministic & isolated?
- Coverage sufficient?

## 4. Security
- No secrets?
- Input validation correct?
- No injection risk?
- No risky dependencies?

## 5. Performance
- No wasteful loops?
- Data structures appropriate?
- No unnecessary I/O?

## 6. Standards & Architecture
- Follows repository conventions?
- No dead code?
- No commented blocks?
- Correct folder placement?

## 7. Suggestions
Provide actionable, concise suggestions with filenames/line numbers.
